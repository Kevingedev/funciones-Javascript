PARÁMETRO REST

Los parámetros REST permiten a una función recibir un número indefinido de argumento como un array.
Se definen usando tres puntos (...) antes del nombre del parámetro. 

            function sumarTodo(...numeros) {
                return numeros.reduce((acumulador, numero) => acumulador + numero, 0);
            }

            function sumar(){
                console.log(sumarTodo(1, 2, 3));
                console.log(sumarTodo(10, 20, 30, 40));
            }

En este ejemplo, números se convierte en un array que contiene todos los argumentos pasados a la función.

Los parámetros rest son aquellos a los que no se les ha asignado un nombre. Son instancias de Array, lo que significa que los métodos como sort, mapforeach o pop pueden aplicarse directamente.
Estos parámetros rest pueden ser desestructurados, eso significa que sus datos pueden ser desempaquetados dentro de distintas variables.

Antes de los parámetros rest se utilizaba arguments, pero ya casi ni se usa, ya que aparte de que arguments no es un array, no podemos ejecutar arguments.map (...), por ejemplo.

Los parámetros rest SIEMPRE deben de ir al final.

Por ejemplo: 

                function f(arg1, arg2, ...rest){}

OPERADOR SPREAD

El operador SPREAD es similar a los parámetros rest, también usa ... , pero hace exactamente lo opuesto.

El operador spread (...) permite expandir elementos de arrays y objetos obteniendo sus elementos individuales.
Es extremadamente útil para trabajar con arrays, objeto y funciones, simplificando muchas tareas comunes.
La sintaxis básica de un operador spread es la siguiente: 

            const array = [1, 2, 3];
            const nuevoArray = [...array, 4, 5];
            console.log(nuevoArray); // [1, 2, 3, 4, 5]

En este ejemplo hemos creado un array. A sus elementos, le añadimos 4 y 5. Usamos esto para generar un nuevoArray con los elementos del array original, añadiendo nuevos elementos al final.

    Uso del operador SPREAD con arrays:

  • Expandir un array:

    • Uno de los usos más comunes es crear nuevos arrays a partir de arrays existentes.
    Esto es útil para copiar arrays o combinar varios arrays en uno solo. Por ejemplo: 

                    const array1 = [1, 2, 3];
                const array2 = [...array1, 4, 5, 6];

                console.log(array2);

  • Copiar arrays:

    • El operador spread también permite crear copiar superficiales de arrays, lo que es útil para evitar efectos secundarios en operaciones de manipulación de datos:

                const original = [1, 2, 3];
            const copia = [...original];
            console.log(copia);
  • Agregar elementos arrays:

    • Puedes utilizarlo para añadir elementos a un array existente sin modificar el array original:

                    const numeros = [1, 2, 3];
                const nuevoNumero = 4;
                const actualizado = [...numeros, nuevoNumero];
                console.log(actualizado);
  • Concatenar arrays:

    • De igual forma, podemos concatenar de forma sencilla dos arrays:

                const array3 = [1, 2, 3];
            const array4 = [4, 5, 6];
            const arrayConcatenado = [...array3, ...array4];

            console.log(arrayConcatenado);
  • Manipulación de dos arrays anidados:

    • El operador spread también puede ser útil para trabajar con arrays anidados:

                const matrices = [[1, 2], [3, 4]];
            const combinada = [].concat(...matrices);
            console.log(combinada);
  • Pasar argumentos a una función:

    • En este ejemplo, el operador spread se utiliza para pasar los elementos del array numeros como argumentos individuales a la función sumar. 

                    function sumar(a, b, c) {
                return a + b + c;
                }
                
                const numbers = [1, 2, 3];
                
                const resultado = sumar(...numbers);
                
                console.log(resultado);
  • Desestructuración y spread:

    • En combinación con la Desestructuración, el operador spread permite extraer parte de un objeto y combinarlo con otros valores.

                    const persona = { nombre: 'Luis', edad: 28, profesion: 'Ingeniero' };
                
                const { nombre, ...resto } = persona;
                
                console.log(nombre); 
                console.log(resto);

El operador spread puede tener implicaciones de rendimiento en operaciones con grandes cantidades de datos, ya que crea copias superficiales de arrays y objetos.

Es recomendable utilizarlo con cuidado en situaciones donde el rendimiento sea una preocupación crítica.

En resumen:

Cuando veamos "..." en el código, son los parámetro rest o el operador spread.

    ¿Cómo se distinguen?

  • Cuando ... se encuenta al final de los parámetros de una función, son los "parámetros rest" y recogen el resto de la lista de argumentos en un array.
  • Cuando ... está en el llamado de una función o similar, se llama "operador spread" y expande un array en una lista.

    • Patrones de uso:

  • Parámetros Rest, son usados para crear funciones que acepten cualquier número de argumentos.
  • Operador spread, es usado para pasar un array a funciones que normalmente requeieren una lista de muchos argumentos.

    • Ambos ayudan a ir entre una lista y un array de parámetros con facilidad.
